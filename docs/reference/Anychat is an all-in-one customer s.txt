By integrating with various third-party platforms such as social media, live chat, and messaging apps, the omnichannel inbox centralizes your incoming messages. This feature is particularly useful for businesses and customer service teams, as it enhances efficiency, improves response times, and helps maintain consistent communication across multiple channels. Chat flow basics
Let's explore the fundamental components (actions) that you can use to create powerful and customizable chatbots.

Event triggers
Triggers enable you to initiate specific actions based on user interactions with the chat widget. For example, an event trigger can act as the starting point for your flow. However, please be aware that many integrations do not support certain event triggers due to the nature of the application. For instance, most integrations do not support widget-related events such as "Chat window opened," "Chat window closed," "Before start new thread," or "Custom event," as these events are based on chat widget behaviors.
Supported events
First Visit - This trigger activates when a customer visits your website for the first time
Second Visit - This trigger activates when a customer has visited your website previously.
Chat Window Opened - This trigger activates when a customer opens the chat window.
Chat Window Closed - This trigger activates when a customer closes the chat window.
Bot Menu Opened - This trigger activates when a customer opens the bot menu.
Before start new thread - This trigger activates when a thread has not yet started or the current thread has ended.
New Thread Started - This trigger activates when a customer starts a new thread by sending the first message or command to the chat.
Custom Event - This trigger can be activated via JavaScript code (see instructions below).

Each trigger has additional options:
Run once per thread: If enabled, this trigger will execute only once per thread.
Run only if thread is not started: If enabled, this trigger will execute only if the thread has not yet started (i.e., no messages have been sent by the website visitor).
Custom Event trigger
You can activate a custom trigger whenever necessary by executing the following JavaScript code:
anychat.sendCustomEvent('myCustomTrigger');

Bot commands
Bot commands originate from the Telegram bot system, but you can also utilize them with our native live chat widget.

In the Telegram app, the bot command is the sole method to initiate a bot. When a user visits your bot in Telegram, they will see the Start button, which sends a /start command when pressed. Therefore, designing a chat flow for Telegram necessitates including the /start command.

If you use bot commands within the live chat widget, they will function identically to those in the Telegram app. The initial command will serve as the starting point, so when a user opens the chat widget, they will see the Start button. Pressing this button will initiate the thread.

Message
The "Message" action allows you to send a message to the customer. This message can include one or several images and may also contain buttons.

Additionally, you can trigger several actions within the "Message" action:
Send a webhook
Set visitor data (store user email, name, and phone number)
Send data to a Google Sheet (requires Google Sheet integration)
Send an email
Send a Telegram notification
Change the chat flow language

If you are expecting a response from the customer, you need to enable the "Wait for reply" option. You can also set validation for the input value, such as requiring email validation if you are expecting an email address. Please see the next article for details.

Handling user input
If you anticipate a response from the customer, you need to enable the "Wait for reply" option in the "Message" action. 
Input validation
You can then optionally set a validation for the input value. For example, if you expect an email address in response to the message, you can set "Email" as the validation. Once validation is configured, you can attach actions to execute if the validation fails (using the "Message is invalid" port). If the input is valid, the action connected to the "Next" port will be triggered.
Saving input value to a variable
To store the value of user input, enable the "Save to variable" option located in the Actions section, and then choose the variable where the input value will be stored.

Forms
Please note that forms are supported only on native live chat and not on other channels. If you are using forms in your chat flow, you will need to implement some workaround logic using the "Condition" action to run a different flow subtree for platforms that do not support forms.
You can send a form directly to the live chat widget. Optionally, you can disable the main input field for the Form action to prevent sending messages until the form is submitted.

Additionally, you can save each form field value to a variable. To do this, enable the "Save response to variable" option in the Actions section. Then, select the variable to store and enter {fieldID} in the Data field.

CRM
CRM stands for Customer Relationship Management. It refers to the strategies, practices, and technologies that companies use to manage and analyze customer interactions and data throughout the customer lifecycle. The goal of CRM is to improve business relationships with customers, aid in customer retention, and drive sales growth.

Our CRM main components:
1. Contact Management:
Details Storage: Store and manage detailed information about each contact, including names, phone numbers, email addresses, job titles, and personal notes.
Interaction Tracking: Maintain a history of communications and interactions with each contact, such as emails, phone calls, meetings, and notes.
Segmentation: Group contacts into segments based on tags.
Search and Filters: Easily search and filter contacts to quickly find specific individuals or groups.
2. Organizations Management:
Company Profiles: Maintain detailed profiles of the organizations your business interacts with, including their industry, size, contact information, and related contacts.
Relationship Mapping: Manage the relationships between different contacts and organizations.
Notes and Attachments: Attach relevant documents, contracts, and notes to organization profiles for easy access.
3. Tickets Management:
Issue Tracking: Create and manage tickets for customer support issues, product defects, or service requests.
Prioritization: Assign priority levels to tickets to ensure that the most critical issues are addressed promptly.
Assignment and Workflow: Assign tickets to specific team members and track the progress through different stages of resolution.
Customer Communication: Maintain clear communication with customers regarding the status of their tickets and resolutions.
4. Tasks Management:
Task Creation: Create tasks for team members, with details such as due dates, priority levels, descriptions, and related contacts or organizations.
Assignment: Assign tasks to specific team members and track their progress.
Collaboration: Enable team members to collaborate on tasks, share updates, and leave comments.
5. Meetings Planning:
Scheduling: Schedule meetings with contacts, including setting dates, times, and locations.
Follow-ups: Create follow-up tasks or tickets based on meeting outcomes to ensure that action items are addressed.

These components work together to provide a comprehensive solution for managing customer relationships, improving efficiency, and enhancing customer satisfaction.

With the custom domains feature, you can link your own domain to replace anychat.one. There are several domains you can connect. Let's explore how to use them.
Frontend domain
The frontend domain has now taken over the functions of login and dashboard, replacing the previous anychat.one domain. You have the flexibility to connect either the root domain or a subdomain as your frontend domain.
To connect a subdomain as Frontend domain to anychat.one, create a CNAME record pointing to "hosting.anychat.one". For connecting a root domain, use an "A" record pointing to the IP address "157.90.227.37".
API domain
The API domain replace api.anychat.one domain in the widget script code and API calls.
To connect a subdomain as API domain to anychat.one, create a CNAME record pointing to "api.anychat.one". For connecting a root domain, use an "A" record pointing to the IP address "157.90.227.37".
Knowledge base hosting domain
The knowledge base hosting domain serves as the hosting platform for your client's standalone domains. When a client chooses to link their own standalone domain to the knowledge base, they need to use a CNAME record to point it to your hosting domain.
If you don't have a Knowledge base hosting domain connected, your clients won't be able to connect a standalone domain to the knowledge base.
To connect a subdomain as API domain to anychat.one, create a CNAME record pointing to "kb.anychat.one". For connecting a root domain, use an "A" record pointing to the IP address "157.90.227.37".
Knowledge base wildcard domain
A wildcard domain for the knowledge base is utilized for your clients' subdomains. Therefore, when a client wishes to claim a specific subdomain for their knowledge base, it will appear as SUBDOMAIN.YOUR_WILDCARD_DOMAIN.
If the Knowledge base wildcard domain is not connected, your clients will be unable to link subdomains to the knowledge base.
To connect your domain as a Knowledge Base wildcard domain, you'll need to verify it by adding a TXT record. During the domain addition process, you'll be provided with the details of the TXT record that you need to create for your domain.
Once validation is complete, you'll need to direct all the subdomains of your KB wildcard domain to our IP address, "157.90.227.37". You can accomplish this by using an "A" record:
go to your DNS settings
Create an "A" record.
In the A record domain set "*." so the resulting domain must be "*.YOUR_WILDCARD_DOMAIN"
In the A record Destination field, enter "157.90.227.37".
Save Changes
Use case
Most of our clients use custom domains in the following way:
create a landing page on the root domain (mychat.com) and place some link to app.mychat.com for customers to login or sign up.
connect the app.mychat.com to anychat as Frontend domain.
connect the api.mychat.com to anychat as API domain.
connect the kb.mychat.com as Knowledge base hosting domain.
connect mychat.com as Knowledge base wildcard domain.
Result
When visitors arrive at the landing page, they have the option to sign up for the dashboard by clicking a link provided there.
When clients want to install the AnyChat widget on their website, they will notice that the embed widget script code includes api.mychat.com (instead of api.anychat.one).
When clients wish to link a free subdomain, such as "docs," to their Knowledge Base, the resulting domain will be docs.mychat.com (instead of docs.anychat.one).
When clients want to connect their custom domain to the knowledge base, they should configure their domain's CNAME record to point to kb.mychat.com (instead of kb.anychat.one).
With this approach, your clients will never encounter the default domains or branding associated with anychat.one.

Standalone live chat widget: 
Open a widget: anychat.showChatPopup();
Close a widget: anychat.hideChatPopup();
Update user info: anychat.updateUserData("Name", "email@example.com");
Send a message: anychat.sendMessage('Message content');
Set custom fields: anychat.setCustomFields({
  'field-slug': 'Field value',
  'field-slug-2': 'Field value'
})
